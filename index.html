Stap 6

Bronze: GeoJSON inladen als Sedona views

We parsen GeoJSON zelf in Python, omdat Fabric de geojson-reader kan onderscheppen met een ESRI GeoAnalytics licentie-error.

import json

def load_geojson(name, srid=28992):
    with open(f"/lakehouse/default/Files/bronze/{name}.geojson") as f:
        data = json.load(f)

    rows = []
    for feat in data["features"]:
        row = {k: (str(v) if v is not None else None)
               for k, v in feat["properties"].items()}
        row["geometry_json"] = json.dumps(feat["geometry"])
        rows.append(row)

    df = spark.createDataFrame(rows)
    df.createOrReplaceTempView(f"raw_{name}")

    cols = [c for c in df.columns if c != "geometry_json"]
    col_list = ", ".join(f"`{c}`" for c in cols)

    sedona.sql(f"""
        CREATE OR REPLACE TEMP VIEW bronze_{name} AS
        SELECT {col_list},
               ST_SetSRID(ST_GeomFromGeoJSON(geometry_json), {srid}) AS geom
        FROM raw_{name}
    """)

for name in ["provincies", "buurten", "stations"]:
    load_geojson(name)
Stap 7

Silver: filteren op Flevoland en opschonen

In Silver maken we de ruwe Bronze-data bruikbaar voor analyse. Eerst halen we de provinciegrens van Flevoland uit de provincielaag. Die grens gebruiken we daarna als ruimtelijk filter voor buurten en stations met ST_Intersects.

  • Waarom dit nodig is: Bronze bevat heel Nederland; de workshopcasus gaat alleen over Flevoland.
  • Wat er gebeurt met buurten: alleen buurten die Flevoland raken blijven over.
  • Datakwaliteit: negatieve CBS-sentinelwaarden worden omgezet naar NULL, zodat ze niet meetellen als echte waarden.
  • Wat er gebeurt met stations: alleen stations binnen Flevoland blijven beschikbaar voor de OV-analyse.
Stap 8

Gold: OV-score en woningverdeling

In Gold maken we analyseklare uitkomsten voor besluitvorming. Per buurt berekenen we de afstand tot het dichtstbijzijnde station, maken we OV-bufferzones voor de kaart en verdelen we de woningopgave van 100.000 woningen over buurten.

  • Afstand tot station: ST_Distance bepaalt de minimale afstand tussen buurtgeometrie en stations.
  • OV-zones: ST_Buffer maakt 500m-, 1000m- en 2000m-zones rond stations voor Fabric Maps.
  • OV-score: buurten dichter bij een station krijgen een hogere score van 1 tot 5.
  • Woningverdeling: de demo verdeelt 100.000 woningen gewogen op ov_score * woningvoorraad.

# Stap 7 en 8 - Silver en Gold voor Woningplanning Provincie Flevoland
# Voer dit uit nadat de Bronze views zijn aangemaakt:
# bronze_provincies, bronze_buurten en bronze_stations.

# Stap 7a - Silver: provinciegrens van Flevoland
sedona.sql("""
CREATE OR REPLACE TEMP VIEW silver_flevoland AS
SELECT geom
FROM bronze_provincies
WHERE naam = 'Flevoland'
""")

# Stap 7b - Silver: buurten binnen Flevoland met CBS-sentinel-opschoning
sedona.sql("""
CREATE OR REPLACE TEMP VIEW silver_buurten AS
SELECT
    b.buurtcode,
    b.buurtnaam,
    b.gemeentenaam,
    CASE WHEN CAST(b.aantalInwoners AS INT) < 0 THEN NULL
         ELSE CAST(b.aantalInwoners AS INT) END AS aantalInwoners,
    CASE WHEN CAST(b.woningvoorraad AS INT) < 0 THEN NULL
         ELSE CAST(b.woningvoorraad AS INT) END AS woningvoorraad,
    CASE WHEN CAST(b.percentageLeegstandWoningen AS DOUBLE) < 0 THEN NULL
         ELSE CAST(b.percentageLeegstandWoningen AS DOUBLE) END AS percentageLeegstandWoningen,
    b.geom
FROM bronze_buurten b
JOIN silver_flevoland f ON ST_Intersects(b.geom, f.geom)
""")

# Stap 7c - Silver: stations binnen Flevoland
sedona.sql("""
CREATE OR REPLACE TEMP VIEW silver_stations AS
SELECT s.*
FROM bronze_stations s
JOIN silver_flevoland f ON ST_Intersects(s.geom, f.geom)
""")

# Stap 8a - Gold: afstand tot dichtstbijzijnde station per buurt
sedona.sql("""
CREATE OR REPLACE TEMP VIEW gold_buurt_score AS
SELECT
    b.buurtcode, b.buurtnaam, b.gemeentenaam,
    b.aantalInwoners, b.woningvoorraad, b.percentageLeegstandWoningen,
    MIN(ST_Distance(b.geom, s.geom)) AS min_afstand_station_m,
    b.geom
FROM silver_buurten b
CROSS JOIN silver_stations s
GROUP BY b.buurtcode, b.buurtnaam, b.gemeentenaam,
         b.aantalInwoners, b.woningvoorraad, b.percentageLeegstandWoningen,
         b.geom
""")

# Stap 8b - Gold: OV-bufferzones voor de visualisatie
sedona.sql("""
CREATE OR REPLACE TEMP VIEW gold_ov_zones AS
SELECT
    s.naam AS station,
    afstand,
    ST_Buffer(s.geom, afstand) AS geom
FROM silver_stations s
LATERAL VIEW explode(array(500, 1000, 2000)) AS afstand
""")

# Stap 8c - Gold: 100.000 woningen verdelen over buurten
sedona.sql("""
CREATE OR REPLACE TEMP VIEW gold_woningopgave AS
WITH scored AS (
    SELECT
        bs.buurtcode, bs.buurtnaam, bs.gemeentenaam,
        bs.aantalInwoners AS aantal_inwoners,
        bs.woningvoorraad,
        bs.percentageLeegstandWoningen AS pct_leegstand,
        bs.min_afstand_station_m,
        CASE WHEN bs.min_afstand_station_m < 500  THEN 5
             WHEN bs.min_afstand_station_m < 1000 THEN 4
             WHEN bs.min_afstand_station_m < 2000 THEN 3
             WHEN bs.min_afstand_station_m < 5000 THEN 2
             ELSE 1 END AS ov_score,
        bs.geom
    FROM gold_buurt_score bs
    WHERE bs.woningvoorraad IS NOT NULL
      AND bs.woningvoorraad > 0
),
gewogen AS (
    SELECT *, (ov_score * woningvoorraad) AS gewicht FROM scored
),
totaal AS (
    SELECT SUM(gewicht) AS som FROM gewogen
)
SELECT
    g.buurtcode, g.buurtnaam, g.gemeentenaam,
    g.aantal_inwoners, g.woningvoorraad, g.pct_leegstand,
    g.min_afstand_station_m, g.ov_score,
    CAST(ROUND(100000.0 * g.gewicht / t.som) AS INT) AS te_bouwen_woningen,
    g.geom
FROM gewogen g CROSS JOIN totaal t
""")
Stap 9

Materialiseren als Delta-tabellen

De Data Agent leest Delta-tabellen, geen temp views. De Sedona geom-kolom gaat eruit.

def materialize(view, table):
    schema_df = sedona.sql(f"SELECT * FROM {view} LIMIT 0")
    cols = [c for c in schema_df.columns if c != "geom"]
    col_list = ", ".join(f"`{c}`" for c in cols)
    sedona.sql(f"SELECT {col_list} FROM {view}") \
        .write.mode("overwrite") \
        .option("overwriteSchema", "true") \
        .saveAsTable(table)

materialize("gold_buurt_score",  "gold_buurt_score")
materialize("gold_woningopgave", "gold_woningopgave")

Exporteren naar GeoJSON voor Fabric Maps

Fabric Maps leest GeoJSON uit Lakehouse Files. De export transformeert EPSG:28992 naar EPSG:4326.

import os
os.makedirs("/lakehouse/default/Files/maps", exist_ok=True)

def export_to_geojson(view, filename):
    schema_df = sedona.sql(f"SELECT * FROM {view} LIMIT 0")
    cols = [c for c in schema_df.columns if c != "geom"]
    col_list = ", ".join(f"`{c}`" for c in cols) if cols else ""
    select_cols = f"{col_list}, " if col_list else ""

    df = sedona.sql(f"""
        SELECT {select_cols}
               ST_AsGeoJSON(ST_Transform(geom, 'EPSG:28992', 'EPSG:4326')) AS geometry_json
        FROM {view}
    """).toPandas()

    features = []
    for _, row in df.iterrows():
        props = {c: row[c] for c in df.columns if c != "geometry_json"}
        features.append({
            "type": "Feature",
            "properties": props,
            "geometry": json.loads(row["geometry_json"])
        })

    feature_collection = {"type": "FeatureCollection", "features": features}

    path = f"/lakehouse/default/Files/maps/{filename}.geojson"
    with open(path, "w") as f:
        json.dump(feature_collection, f)
    print(f"Geschreven: {path}  ({len(features)} features)")

export_to_geojson("silver_flevoland",   "flevoland")
export_to_geojson("silver_stations",    "stations")
export_to_geojson("gold_ov_zones",      "ov_zones")
export_to_geojson("gold_woningopgave",  "buurt_score")

In dit deel bouwt de data-engineer een complete geo-analytische oplossing: van open PDOK-data tot een interactieve kaart en een Nederlandstalige Data Agent voor strategische vragen over de Woondeal Flevoland: 100.000 nieuwe woningen voor 2030.

WFS-bronnen voor Deel II

Deze links zijn de start van de Flevoland-pipeline. WFS levert echte features met geometrie en attributen; anders dan WMS is het dus geschikt om in een notebook op te halen, te filteren, te verrijken en als Bronze-data vast te leggen.

Wat houdt WFS in?

WFS staat voor Web Feature Service. Waar WMS vooral een kaartbeeld teruggeeft, geeft WFS objecten terug: bijvoorbeeld provinciegrenzen, buurten of stations. Daardoor kun je ruimtelijke berekeningen doen, zoals filteren op Flevoland, afstanden tot stations bepalen en buurten scoren voor woningbouw.

Stap 2

Bronze: open data ophalen van PDOK

Drie GeoJSON-bestanden landen in Files/bronze/: provincies, buurten en stations.

import json

def load_geojson(name, srid=28992):
    with open(f"/lakehouse/default/Files/bronze/{name}.geojson") as f:
        data = json.load(f)

    rows = [
        {
            **{
                k: str(v) if v is not None else None
                for k, v in feat["properties"].items()
            },
            "geometry_json": json.dumps(feat["geometry"])
        }
        for feat in data["features"]
    ]

    df = spark.createDataFrame(rows)
    df.createOrReplaceTempView(f"raw_{name}")

    cols = ", ".join(f"`{c}`" for c in df.columns if c != "geometry_json")
    sedona.sql(f"""
        CREATE OR REPLACE TEMP VIEW bronze_{name} AS
        SELECT {cols},
               ST_SetSRID(ST_GeomFromGeoJSON(geometry_json), {srid}) AS geom
        FROM raw_{name}
    """)

load_geojson("provincies")
load_geojson("buurten")
load_geojson("stations")
Stap 3

Silver: opschonen en filteren tot Flevoland

Bronze bevat heel Nederland. Silver beperkt het studiegebied tot Flevoland met één ruimtelijke join.

CREATE OR REPLACE TEMP VIEW silver_flevoland AS
SELECT geom
FROM bronze_provincies
WHERE naam = 'Flevoland';

CREATE OR REPLACE TEMP VIEW silver_buurten AS
SELECT b.*
FROM bronze_buurten b
JOIN silver_flevoland f
  ON ST_Intersects(b.geom, f.geom);

CREATE OR REPLACE TEMP VIEW silver_stations AS
SELECT s.*
FROM bronze_stations s
JOIN silver_flevoland f
  ON ST_Intersects(s.geom, f.geom);
Stap 4

Gold: OV-bereikbaarheid berekenen per buurt

Voor elke buurt bereken je afstand tot het dichtstbijzijnde station en een OV-score van 1 tot 5.

CREATE OR REPLACE TEMP VIEW gold_buurt_score AS
SELECT b.*,
       MIN(ST_Distance(b.geom, s.geom)) AS min_afstand_station_m
FROM silver_buurten b
CROSS JOIN silver_stations s
GROUP BY <alle kolommen van b>;

Voeg daarnaast gold_ov_zones toe met buffers van 500m, 1000m en 2000m rond stations voor de kaartvisualisatie.

Stap 5

Gold: woningverdeling berekenen

Verdeel de 100.000 woningen uit de Woondeal over buurten, gewogen op OV-score en woningvoorraad.

gewicht = ov_score * woningvoorraad
te_bouwen_woningen = ROUND(100000 * gewicht / SUM(gewicht))

Governance-keuze: CBS gebruikt negatieve sentinel-waarden voor geheime cijfers in kleine buurten. Die worden niet hard in Silver verwijderd, maar expliciet geïnstrueerd aan de Data Agent.

Stap 6

Materialiseren als Delta-tabellen

Temp views zijn niet genoeg: de Data Agent leest Delta-tabellen. De Sedona geom-kolom wordt weggelaten.

def materialize(view, table):
    cols = [
        c for c in sedona.sql(f"SELECT * FROM {view} LIMIT 0").columns
        if c != "geom"
    ]
    col_list = ", ".join(f"`{c}`" for c in cols)

    sedona.sql(f"SELECT {col_list} FROM {view}") \
        .write.mode("overwrite") \
        .option("overwriteSchema", "true") \
        .saveAsTable(table)

materialize("gold_buurt_score", "gold_buurt_score")
Stap 7

Exporteren naar GeoJSON voor Fabric Maps

Fabric Maps gebruikt GeoJSON uit Lakehouse Files. Houd minder dan 200 properties per feature aan en transformeer naar WGS84.

ST_AsGeoJSON(ST_Transform(geom, 'EPSG:28992', 'EPSG:4326'))
Waarom slim exporteren?De ruwe buurtenlaag heeft 200+ CBS-kolommen. Fabric Maps heeft een limiet van 200 properties per feature.
Waarom transformeren?PDOK-data staat vaak in EPSG:28992. Fabric Maps werkt met latitude/longitude in EPSG:4326.
